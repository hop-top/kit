## User story

**As a** `kit-contributor`, **I want** CI to use the same Docker
image as my local dev container, **so that** build failures are
reproducible locally and there's no environment drift between dev
and CI.

## Acceptance criteria

- [ ] `.github/actions/dev-env/` composite action builds same
  Dockerfile
- [ ] Docker layers cached via GHA cache
- [ ] Nix builder stage rebuilds only when flake files change
- [ ] No AI tools installed in CI runs
- [ ] `task check` passes inside CI container
