## User story

**As a** `cli-author`, **I want** to attach images, audio, and video
to LLM requests using a provider-agnostic API, **so that** my tool
works across OpenAI, Anthropic, and Ollama without adapter-specific
code.

## Acceptance criteria

- [x] `Message.Parts []ContentPart` accepted by `Complete`/`Stream`
- [x] `Message.Content` (text-only) still works unchanged
- [x] `InlineSource`, `URLSource`, `FileSource` implement `MediaSource`
- [x] `Client.GenerateImage` with fallback + hooks
- [x] `Client.Transcribe` with fallback + hooks
- [x] `Client.Synthesize` with fallback + hooks
- [x] `ErrUnsupportedModality`, `ErrMediaTooLarge`, `ErrInvalidFormat`
  all non-fallbackable
- [x] `client.Capabilities()` reports `image_gen`, `transcribe`,
  `synthesize`, `video_analyze`, `video_gen`
- [x] OpenAI adapter: multimodal input, image gen, transcribe, tts
- [x] Anthropic adapter: multimodal input
- [x] Ollama adapter: multimodal input, image gen (via generate API)

## Notes

Phase 3 of llm-multimodal track. Adapters implement new interfaces
via type assertion; callers that don't need multimodal see no change.
