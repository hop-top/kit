# Release Checklist

Pre-release checklist for kit maintainers. For full details, see
[RELEASING.md](../RELEASING.md).

## Pre-release

- [ ] CI green on `main`
- [ ] `make lint` passes locally (Go + TS + Python + docs)
- [ ] `go test ./...` passes
- [ ] `cd ts && pnpm test` passes
- [ ] `cd sdk/py && uv run pytest` passes
- [ ] `go test -tags parity ./cli/...` passes (parity tests)
- [ ] No stale tracks/tasks blocking release
- [ ] CHANGELOG reviewed (auto-generated by release-please;
      see `release-please-config.json` for section config)
- [ ] `make test-release` passes (e2e release script tests)

## Release process

1. Conventional commits on `main` trigger release-please
2. release-please creates/updates a release PR
   (`release-please--branches--main`)
3. PR merges via:
   - **Nightly auto-merge** (04:00 UTC) if CI green
     (`nightly-release.yml`); or
   - **Manual merge** by maintainer at any time
4. Merged release PR triggers:
   - GitHub Releases + tags (one per language)
   - goreleaser builds cross-platform Go binaries
   - npm publish (`@hop-top/kit`, requires `NPM_TOKEN` secret)
   - PyPI publish (`hop-top-kit`, trusted publishing via OIDC)

### Version synchronization

Go, TS, Python share major.minor via `linked-versions` in
`release-please-config.json`. Patch may differ.

## Post-release

- [ ] Verify Go binaries on GitHub Release page
- [ ] Verify `go get hop.top/kit@latest` resolves new version
- [ ] Verify npm: `npm view @hop-top/kit version`
- [ ] Verify PyPI: `pip index versions hop-top-kit`
- [ ] Update downstream deps (tlc, aps, rux, etc.)
- [ ] Announce if major features shipped

## Promotion workflow

Stage promotion gates releases through maturity levels.
See [RELEASING.md](../RELEASING.md) for transition criteria
and audience definitions.

```
alpha -> beta -> rc -> release
```

Interactive:

```bash
make promote
```

Explicit:

```bash
make promote-alpha      # start new version cycle
make promote-beta       # feature-complete
make promote-rc         # no known blockers
make promote-release    # 7-day bake passed, no regressions
```

Each promotion:
- [ ] Validates transition order
  (`scripts/promote-release.sh`)
- [ ] Updates `prerelease-type` in all packages
  (`release-please-config.json`)
- [ ] Commits change; PR triggers CI gate
  (`release-promote-gate.yml`)
- [ ] Requires CODEOWNERS approval (repo admins / release
      team)

## Nightly auto-release

Cron at 04:00 UTC (`nightly-release.yml`). Disable by setting
`NIGHTLY_RELEASE` repo variable to `false`.
